Архітектура шаблону. Файли та папки

Категорії: Документація «Чертоги Фрілансера» v3.0.0

У цьому розділі поговоримо про архітектуру, ієрархію папок та файлів, стартового шаблона «Чертоги Фрілансера» v3.0.0 (далі ЧФ)

Розархівувавши завантажений шаблон, в корені ви виявите такі файли та папки:

  1. Файл package.json – файл, що містить команди запуску збирачів, інформацію про встановлені плагіни та їх версії (можна редагувати за потреби). А також іншу інформацію про автора та версію шаблону.
  2. Файл gulpfile.js – головний файл збирача GULP. Містить підключення різних завдань та сценарії їх виконання (можна редагувати за потреби)
  3. Файл snnipets.txt – файл зі сніппетами, які використовуються при роботі з шаблоном. Як їх додати до свого редактора описано тут
  4. Файл README.txt – коротка інформація щодо роботи з шаблоном
  5. Файл cover.jpg – обкладинка ЧФ
  6. Папка config – містить папки та файли для налаштування роботи інструментів, а також файли з окремими завданнями. Докладніше..
  7. Папка src – містить вихідні файли та папки проекту. Докладніше..

Вміст папки config:

  1. Файл gulp-settings.js – містить налаштування шляхів до файлів та папок для роботи GULP, а також налаштування FTP з’єднання для вивантаження результату на сервер
  2. Файл gulp-plugins.js – допоміжний файл, містить підключення та експорт списку загальних плагінів для завдань GULP
  3. Файл webpack.dev.js – містить конфігурацію роботи WEBPACK у режимі розробника
  4. Файл webpack.prod.js – містить конфігурацію роботи WEBPACK у режимі продакшн
  5. Папка gulp-tasks – містить файли окремих завдань GULP

Вміст папки src:

  1. Файл favicon.ico – іконка сайту
  2. Файл index.html -індексна сторінка-зміст, вона не оновлюється автоматично під час редагування (за бажанням видалити)
  3. Файл home.html – головна сторінка сайту (за бажанням перейменувати в index.html)
  4. Папка files (порожня) –  всі файли цієї папки будуть скопійовані в папку з результатом (dist/files)
  5. Папка fonts (порожня) – використовується для підключення локальних та іконкових шрифтів
  6. Папка svgicons (порожня) – використовується для створення SVG спрайту
  7. Папка html – містить .HTM файли, які підключаються до HTML файлів сторінок (наприклад, home.html). Тут можна створювати свої файли, що підключаються
  8. Папка scss – містить файли та папки стилів сайту
  9. Папка js – містить файли та папки скриптів сайту
  10. Папка img – тут зберігаються зображення проекту. Спочатку тільки кавер-фото шаблону

Вміст папки src/html:

Файли цієї папки призначені для підключення до HTML-файлів сторінок.

  1. Файл _header.htm – підготовка до створення шапки сайту. Містить тег <header> та обмежуючий контейнер. Спочатку підключено до home.html
  2. Файл _footer.htm – підготовка до створення підвалу сайту. Містить тег <footer> та обмежуючий контейнер. Спочатку підключено до home.html
  3. Файл _head.htm – містить тег <head> всередині якого вказані метатеги, підключена іконка сайту, файл стилів, а також тег <title> зі змінною-заголовком. Спочатку підключено до home.html
  4. Файл _js.htm – призначений для підключення загальних JS-файлів. Містить підключення основного файлу js/app.min.js із результату. Спочатку підключено до home.html
  5. Файл _popup.htm – підготовка до створення попапов. Містить шаблонний закоментований HTML-код. Спочатку підключений до home.html. Дивись роботу з попапами
  6. Файл _pagging.htm – підготовка до створення блоку посторінкової навігації. Містить шаблонний HTML-код.
  7. Файл _preloader.htm – містить HTML CSS та JS функціонал прелоадеру до сайту

Вміст папки src/scss:

  1. Файл style.scss – основний файл стилів. Містить різні налаштування та підключення інших файлів. Докладніше…
  2. Файл home.scss – файл стилів на головній сторінці. Спочатку порожній і підключений до style.scss
  3. Файл header.scss – файл стиль шапки сайту. Містить закоментований код кнопки меню-бургера. Спочатку підключено до style.scss
  4. Файл footer.scss – файл стилів підвалу сайту. Спочатку порожній і підключений до style.scss
  5. Файл common.scss – файл стилів для загальних блоків конкретного проекту, що перевикористовуються. Спочатку порожній і підключений до style.scss
  6. Файл base.scss – файл базового стилю шаблону ЧФ. Містить корисні SCSS-шаблони та закоментовані підключення файлів із базовими стилями різних модулів.
  7. Папка libs – містить файли повних (заводських) стилів різних модулів та плагінів
  8. Папка fonts – містить файл(и) стилів, пов’язаних зі шрифтами. Містить файл icons.scss для підключення іконочного шрифту. Також,, у процесі підключення локальних шрифтів, тут з’являється файл fonts.scss
  9. Папка base – містить допоміжні файли стилів ЧФ, також файли з базовими стилями різних моделей. Для зручності файли модулів однієї групи (наприклад елементи форм) зібрані в окремі підпапки. Тут же знаходиться файл із обнуляючими стилями,  null.scss

Вміст папки src/js:

  1. Файл app.js – основний файл скриптів. Служить для підключення необхідного у проекті функціоналу та інших налаштувань.
  2. Папка libs – містить файли готових доповнень, як правило, не передбачені для редагування.
  3. Папка files – містить файли різних модулів та іншого функціоналу.
Вміст папки src/js/files:
  1. Файл script.js – призначений для написання свого коду для проекту. Спочатку імпортовано до app.js
  2. Файл functions.js – містить дрібні модулі та різний допоміжний функціонал. Спочатку імпортований в app.js, але модулі закоментовані
  3. Файл gallery.js – містить підключення плагіна та стилів галереї. Спочатку імпортований в app.js, закоментований. При необхідності, в нього можна вносити зміни
  4. Файл sliders.js – містить підключення плагіна та стилів слайдера, функціонал для автоматичного додавання класів, код для створення конкретних слайдерів. Спочатку імпортований в app.js, закоментований.
  5. Файл tippy.js – містить підключення плагіна та стилів плагіна підказок. Спочатку імпортований в app.js, закоментований.
  6. Файл map.js (в работі) – містить шаблонний код для реалізації функціоналу карток.
  7. Файл modules.js – допоміжний файл для роботи ЧФ.
  8. Папка forms – містить файли модулів для роботи з формами.
  9. Папка scroll – містить модулі для роботи з прокручуванням сайту.
